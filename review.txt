Version: 2.1.0
Date: 2026-05-27
Author: Awhadi / Codex

Summary:
Sign-In Experience, Security, User Management, and Tools were refined so related settings are grouped together and existing SSO behavior stays intact. Verification overlays now inherit theme styling, include optional provider/aOAUTH branding, and offer Paper Plane, Glass Shield, and Aurora styles. A new self-service `[aoauth_link_account]` shortcode lets logged-in users link an enabled SSO provider when account linking is allowed. Tools now focuses on useful maintenance actions instead of repeating the Users screen.

Security notes:
- Existing public login bot protection remains enforced for normal SSO login.
- Self-service account linking requires a logged-in session, account-linking settings enabled, a user-specific nonce, and matching provider/WordPress email addresses.
- Maintenance actions require manage_options and the existing admin nonce.
- Verification overlay branding is presentation-only and does not alter Turnstile or reCAPTCHA token verification.

UX notes:
- Sign-In Experience now groups login display, button appearance, account linking, and verification overlay controls.
- Verification overlay preview sits beside the settings and updates with selected theme/style.
- Verification overlay color and message-style fields were removed from the UI because colors are now theme-derived.
- Tools now exposes Plugin Debug Logging, WordPress debug status, SSO-linked user count, and focused maintenance actions.

Rollback plan:
Restore the previous v2.0.0 tag, clear object/page caches, and verify a normal SSO login plus one bot-protected login. If settings need rollback, import a pre-release configuration backup from Tools > Backup & Restore.
