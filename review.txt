Summary:
This patch refines the v2.4.0 sign-in and bot-verification UX without changing provider configuration or OAuth callback behavior. Inside Login Form now places SSO buttons after the WordPress Login button, overlay choices are narrowed to Spotlight, Constellation, and Minimal, duplicate layout choices are replaced with clearer useful layouts, overlay previews inherit theme colors, and Full Login Ban now reads its configured duration before setting a ban.

Security/UX notes:
- Inside Login Form now means username/password, Remember Me, Login button, then "Or login with" and SSO buttons.
- Below Login Form remains available for sites that prefer SSO outside the WordPress form.
- Only Spotlight, Constellation, and Minimal are selectable for bot verification overlays.
- Full Login Ban is now applied using the configured duration after repeated failed account-linking password checks.
- Turnstile Display Mode help text explains that the overlay is the screen cover while the mode controls how the Cloudflare widget appears inside it.

Performance notes:
- No new frontend bundles were added.
- Existing login CSS/JS files were reused for placement and overlay behavior.
- Legacy saved layout/overlay values are normalized without extra database work.

Rollback plan:
Restore v2.4.0 if needed. If a site depends on the old selectable overlay styles, save the Sign-In Experience settings again after rollback.
