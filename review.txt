Security/UX notes:
- SSO buttons default below the WordPress login form to reduce accidental provider clicks after password entry.
- Bot verification clearing for users is scoped to the current request/IP data and requires a logged-in user nonce.
- Global bot verification clearing remains administrator-only.
- Managed Turnstile mode renders inside the overlay so timeout/retry states are clearer for visitors.
- Users table SSO actions are merged into the provider column while keeping the existing unlink nonce and administrator safeguards.
- Inline image fallback handlers were removed and replaced with external JavaScript handlers.

Performance notes:
- Log cleanup remains native WP-Cron; Tools now shows next/last cleanup status and manual cleanup controls.
- No new frontend JavaScript or CSS bundles were added.
- Theme icon tinting is CSS-only.
- Existing login, account linking, and unlink scripts are reused for new UI behavior.

Rollback plan:
