Version: 2.2.0
Date: 2026-05-27
Author: Awhadi / Codex

Summary:
User Management was reorganized into clearer groups, and account linking now supports multiple enabled providers on the same WordPress user. The WordPress profile SSO section now shows all connected providers and lets users link additional providers from their own profile. Existing themes were restyled to look visibly different without external fonts, and the Paper Plane overlay was revised so a small plane moves left-to-right while the overlay fills behind it.

Security notes:
- Provider identity is stored per provider using subject when available, with email as fallback.
- A provider identity cannot be linked to more than one WordPress user.
- Provider email may differ from the WordPress email only after the logged-in user completes a nonce-protected OAuth link flow.
- Admins editing another user's profile can view provider status, but cannot initiate OAuth linking for that other user from their own session.
- Legacy `_aoauth_provider` metadata remains as a primary provider for backward compatibility.

UX notes:
- User Management now matches the grouped structure used by the other settings tabs.
- User profiles show multiple connected SSO providers as cards.
- Existing themes now differ in color, sizing, shape, and emphasis while using system fonts.
- Paper Plane overlay now visually communicates motion and screen coverage more clearly.

Rollback plan:
Restore the previous v2.1.0 tag, clear caches, and verify login with the original primary provider. Existing legacy provider metadata remains compatible, but multi-provider links added after this release would not be understood by older versions.
