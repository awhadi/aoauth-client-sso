Security notes:
- OAuth callback failures now redirect with a generic public message while detailed errors remain in debug/log channels.
- Log ordering is restricted to a whitelist of known database columns.
- Single-account unlink AJAX now requires the user-specific nonce and no longer accepts the global admin nonce.
- Full login bans are enforced for SSO login attempts and WordPress password authentication during the ban period.
- Role redirect settings are sanitized to relative paths or safe validated URLs.

UX notes:
- "Login Button Theme" is renamed to "Single Sign-On Theme" because it now controls login, account-linking, and verification overlay presentation.
- User management now includes role redirect inputs for all editable roles.
- Security settings explain the practical impact of High vs Medium security levels.
- Tools now contains debug guidance, log controls, SSO user visibility, backup/restore, and factory reset.
- Logs now include filtering controls.

Rollback plan:
Restore the previous tag, deactivate and reactivate the plugin if needed, then verify provider configuration and login flow. If database settings need rollback, import a configuration backup from Tools > Backup & Restore before re-enabling production SSO.
