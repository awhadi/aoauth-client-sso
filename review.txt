Review scope:
- Security review of OAuth/OIDC login, callback handling, admin AJAX, bot protection, secret storage, import/export, redirects, and account linking.
- Performance review of request-time loading, logging, pagination, and remote provider calls.
- Usability review of admin setup, provider testing, login buttons, and account-linking flows.

Security issues addressed:
- Replaced OAuth state creation with a high-entropy per-login random token.
- Added an OIDC nonce to each login request and verify it on ID token return.
- Added ID token validation for expiration, not-before time, nonce, audience/client ID, issuer, verified email, and RS256 JWKS signatures.
- Added JWKS retrieval and transient caching for provider signing keys.
- Preserved OIDC discovery metadata, including issuer and jwks_uri, in provider configuration.
- Added default OIDC metadata for Google and Apple when older saved configurations do not contain issuer or jwks_uri.
- Changed bot protection from client-side-only gating to server-enforced one-time verification tokens.
- Encrypted Turnstile and reCAPTCHA secret keys before storage.
- Decrypted protected captcha secret keys only when needed for display or verification.
- Removed captcha secrets from exported configuration backups.
- Sanitized imported settings and provider configuration before saving.
- Tightened redirect URL validation with WordPress-safe redirect validation and http/https scheme checks.
- Validated default role settings before saving.
- Clamped rate-limit and account-linking security settings to safe ranges.

Performance issues addressed:
- Cached JWKS signing keys for one hour to avoid repeated remote key fetches on every login.
- Clamped AJAX log pagination limits to avoid excessive admin-ajax load.

Usability notes:
- Existing setup, login, account-linking, provider management, bot protection, import/export, and logging functionality was preserved.
- Provider test behavior is still based on the existing plugin flow so current UX is not removed.
- Existing saved providers without discovery metadata continue to work, but full ID token signature validation requires jwks_uri. Google and Apple are covered with built-in defaults; custom OIDC providers should use discovery or include jwks_uri.
