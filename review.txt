Version: 2.2.1
Date: 2026-05-27
Author: Awhadi / Codex

Summary:
Tools now separates normal Activity Logs from low-level Deep Debug. The log toggle controls database-backed logs shown on the Logs screen, while Deep Debug shows wp-config.php status and the exact constant needed to enable protected debug files. Tools also now includes a shortcode reference so administrators can quickly find the link and unlink shortcodes.

Files changed:
- aoauth-client-sso.php
- CHANGELOG.md
- review.txt
- admin/css/admin-style.css
- admin/views/tools-settings.php
- includes/class-debug.php

Security/UX notes:
- Deep Debug is intentionally controlled from wp-config.php because it can write detailed OAuth/request data to files.
- The plugin now recognizes `define("OAUTH-DEBUG", "enabled");` in addition to the existing `AOAUTH_DEBUG` constant.
- Shortcode documentation is informational only and does not change shortcode behavior.

Rollback plan:
Restore v2.2.0 if needed. Remove `define("OAUTH-DEBUG", "enabled");` from wp-config.php after troubleshooting if it was added.
