Version: 2.3.0
Date: 2026-05-27
Author: Awhadi / Codex

Summary:
Deep Debug can now be toggled from Tools when wp-config.php is writable, while still showing the manual constant for locked-down servers. The Logs screen keeps pagination but no longer performs a duplicate initial AJAX reload after rendering the first page. Three new icon-only themes were added with light/gradient icon surfaces so dark provider logos remain visible. Activation now opens the wizard only when no provider is configured.

Security/UX notes:
- Deep Debug writes to wp-config.php only for administrators and only with a valid admin nonce.
- If wp-config.php is not writable, the UI fails safely and shows the manual constant.
- Deep Debug can expose detailed request flow data in protected upload logs, so it should be disabled after troubleshooting.
- New icon-only themes keep provider logos visible by avoiding dark icon surfaces.

Performance notes:
- Logs are paginated at 50 rows per page.
- The initial Logs page no longer repeats the same query through AJAX after server render.
- Added provider/status/date indexes to the logs table for faster admin filtering.
- Frontend impact is minimal: theme CSS is conditional, public SSO JS remains on login pages, and no new global frontend bundle was added.

Rollback plan:
Restore v2.2.1 if needed. If Deep Debug was enabled, disable it from Tools or remove `define("OAUTH-DEBUG", "enabled");` from wp-config.php.
